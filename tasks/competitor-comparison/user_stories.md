# User Stories: Сравнение бонусов конкурентов

## US-1: Начало нового сравнения

**Как** CRM-менеджер,
**я хочу** выбрать регион и тип промо для нового сравнения,
**чтобы** дальше добавить релевантных конкурентов.

### User Flow
Хаб инструментов → «Сравнение конкурентов» (список сравнений) → «Новое сравнение» → Экран настройки (регион + тип промо) → Экран добавления конкурентов

### Acceptance Criteria
- [ ] Список регионов ограничен уже поддерживаемыми Retomat регионами (EU/CIS/Crypto/Sweep/MN/LatAm)
- [ ] Список типов промо: Бонус, Турнир, Лояльность
- [ ] Нельзя перейти дальше без явного выбора региона и типа промо
- [ ] Значения по умолчанию не предзаполнены

---

## US-2: Поиск бонуса конкурента через ИИ

**Как** CRM-менеджер,
**я хочу** указать название казино-конкурента и попросить ИИ найти его текущее бонусное предложение,
**чтобы** не искать информацию вручную.

### User Flow
Экран добавления конкурентов → ввод названия казино → «Найти через ИИ» → экран загрузки → Карточка найденного конкурента (или сообщение «не найдено»)

### Acceptance Criteria
- [ ] Пользователь вводит название казино текстом, без обязательного URL
- [ ] Во время поиска показан индикатор загрузки
- [ ] При успехе — карточка конкурента с найденными параметрами бонуса, пометкой «Найдено ИИ» и ссылкой на источник, если она доступна
- [ ] При неудаче — понятное сообщение, что данные не найдены, с предложением ввести вручную (переход к US-3)
- [ ] Пользователь может запустить повторный поиск

---

## US-3: Ручной ввод бонуса конкурента

**Как** CRM-менеджер,
**я хочу** вручную вставить условия бонуса конкурента,
**чтобы** сравнить точные данные, если ИИ их не нашёл или я хочу использовать данные из другого источника.

### User Flow
Экран добавления конкурентов → «Добавить вручную» → форма ввода (название казино + текст/поля условий) → Карточка добавленного конкурента

### Acceptance Criteria
- [ ] Форма принимает свободный текст условий бонуса и/или структурированные поля (размер бонуса, вейджер, мин. депозит и т.д. — набор зависит от типа промо)
- [ ] Добавленный вручную конкурент помечен «Введено вручную», без пометки «Найдено ИИ»
- [ ] Пустая форма не может быть сохранена — обязательны минимум название казино и одно условие

---

## US-4: Выбор своего предложения для сравнения

**Как** CRM-менеджер,
**я хочу** выбрать своё сгенерированное предложение (сохранённое или текущий черновик),
**чтобы** сравнить его с конкурентами.

### User Flow
Экран добавления конкурентов → «Выбрать своё предложение» → список сохранённых предложений (отфильтрованный по типу промо из US-1) или «Использовать текущий черновик» → подтверждение выбора

### Acceptance Criteria
- [ ] Список показывает только предложения того типа промо, который выбран в US-1
- [ ] Если сохранённых предложений нет — пустое состояние с подсказкой создать предложение в соответствующем генераторе
- [ ] Гостю (без аккаунта) недоступен список сохранённых предложений, но доступен текущий черновик из открытой вкладки генератора, если он есть
- [ ] Выбор можно изменить до генерации результата сравнения

---

## US-5: Просмотр результата сравнения

**Как** CRM-менеджер,
**я хочу** увидеть таблицу сопоставления и вывод от ИИ,
**чтобы** понять, где моё предложение сильнее или слабее конкурентов.

### User Flow
Экран добавления конкурентов (минимум 1 конкурент + своё предложение выбраны) → «Сравнить» → экран загрузки → Экран результата (таблица + вердикт + рекомендации)

### Acceptance Criteria
- [ ] Таблица показывает ключевые параметры бонуса по каждому конкуренту и по предложению Retomat, набор параметров соответствует типу промо
- [ ] Для каждого значения видно, из какого источника оно получено (ИИ-поиск / ручной ввод / сгенерировано Retomat)
- [ ] Текстовый вердикт указывает минимум одну сильную и одну слабую сторону предложения Retomat относительно конкурентов
- [ ] Есть минимум одна конкретная рекомендация по параметру (например, «снизить вейджер до X, чтобы быть на уровне конкурентов»)
- [ ] Если у конкурента неполные данные, это видно в таблице явно, а не пропадает молча

---

## US-6: Сохранение и повторный просмотр сравнения

**Как** зарегистрированный пользователь,
**я хочу** сохранить сравнение и вернуться к нему позже,
**чтобы** не повторять поиск заново.

### User Flow
Экран результата → «Сохранить» → Список сравнений (с новой записью) → клик по записи → Экран результата (сохранённая версия)

### Acceptance Criteria
- [ ] Сохранённое сравнение доступно из списка «Сравнение конкурентов» с датой создания, регионом и типом промо
- [ ] Открытие сохранённого сравнения показывает данные на момент сохранения, без автоматического повторного поиска
- [ ] На сохранённом сравнении есть кнопка «Обновить данные конкурентов», запускающая новый ИИ-поиск по тем же конкурентам
- [ ] Гость видит предупреждение, что сравнение не будет сохранено между сессиями без аккаунта

---

## US-7: Прозрачность при недостоверных или отсутствующих данных

**Как** CRM-менеджер,
**я хочу** явно видеть, когда ИИ не уверен в найденных данных,
**чтобы** не принимать решения на основе ошибочной информации.

### User Flow
Без UI — сквозной принцип, проявляется в US-2 и US-5 (Экран добавления конкурентов / Экран результата)

### Acceptance Criteria
- [ ] Если ИИ не нашёл официальный источник, значение не показывается как факт — помечается «не подтверждено» или исключается из таблицы
- [ ] Вердикт ИИ никогда не использует неподтверждённые значения как основание для рекомендации без соответствующей оговорки
- [ ] Пользователь всегда может заменить любое найденное ИИ значение на своё, введённое вручную
